A modern, full-stack real-time messaging application built with Next.js 16 (App Router), Convex, Clerk, and Tailwind CSS. 

## 🚀 Features

- **Real-Time Messaging**: Lightning-fast instant messaging powered by Convex WebSockets.
- **Authentication**: Secure user sign-up, sign-in, and session management using Clerk.
- **Rich User Interface**: Beautiful, responsive design using Tailwind CSS v4 and `shadcn/ui`.
- **Presence & Status**: Live online/offline indicators tracked via a robust heartbeat mechanism.
- **Typing Indicators**: Real-time bouncing "Is typing..." animations.
- **Smart Unread Badges**: Tracks unread messages accurately per conversation.
- **Advanced Chat Capabilities**:
  - Soft-delete messages.
  - Emoji reactions on messages.
  - Group chat creation with multiple participants.
- **Smart Scrolling**: Auto-scrolls to the bottom on new messages, and provides a sleek "New messages" indicator when reading history.

## 🛠️ Tech Stack 

- **Frontend**: Next.js 16, React 19, Tailwind CSS v4, `shadcn/ui`, `lucide-react`.
- **Backend & Database**: Convex (Serverless real-time database and functions).
- **Authentication**: Clerk (with JWT templates for seamless Convex integration).

## 💻 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/Irshad-Ahmaed/Real-Time-Chat-Messaging.git
cd Real-Time-Chat-Messaging
```

### 2. Install dependencies

```bash
npm install
# or
yarn install
# or
pnpm install
```

### 3. Set up Environment Variables

Create a `.env.local` file in the root of the project and add your API keys:

```env
# Clerk Authentication Keys
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_publishable_key
CLERK_SECRET_KEY=your_clerk_secret_key

# Clerk Redirect Routes
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_SIGN_IN_FALLBACK_REDIRECT_URL=/chat
NEXT_PUBLIC_CLERK_SIGN_UP_FALLBACK_REDIRECT_URL=/chat

# Convex Deployment URL (Auto-generated by 'npx convex dev')
NEXT_PUBLIC_CONVEX_URL=your_convex_url
```

### 4. Run the Development Servers

You will need two terminal windows open to run this project locally.

**Terminal 1:** Start the Convex backend (this syncs your schema and runs the database locally/on the cloud).
```bash
npx convex dev
```

**Terminal 2:** Start the Next.js frontend development server.
```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## 🗄️ Project Structure

- `app/`: Next.js App Router pages and layouts (including chat routes and auth guards).
- `components/chat/`: Core chat UI components (Sidebar, MessageBubble, ChatArea, etc.).
- `components/ui/`: Reusable `shadcn/ui` components.
- `convex/`: Backend logic including schema, mutations (write), and queries (read).
- `hooks/`: Custom React hooks (`useOnlineStatus`, `useTypingIndicator`, `useSmartScroll`).
- `lib/`: Utility functions (like timestamp formatters).

## 🔒 Webhook Configuration (Clerk)

For user profiles to sync properly, you must set up a Webhook in the Clerk Dashboard pointing to your Convex HTTP action:
1. Go to Clerk Dashboard -> Webhooks.
2. Add Endpoint: `https://<YOUR_CONVEX_URL>.convex.site/clerk`
3. Subscribe to: `user.created`, `user.updated`, `user.deleted`.
4. Copy the Webhook Secret and add it to your Convex environment variables in the Convex Dashboard as `CLERK_WEBHOOK_SECRET`.
